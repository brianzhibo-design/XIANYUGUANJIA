# 闲管家接口归一化文档模板

## 1. 用途
- 描述本接口在业务中的作用与适用场景。
- 说明调用方、被调用方与关键业务目标。

## 2. 请求路径方法
- **Path**：`<填写接口路径>`
- **Method**：`GET | POST | PUT | DELETE`（按实际保留）
- **Content-Type**：`application/json`（按实际填写）

## 3. 请求字段
| 字段名 | 类型 | 必填 | 说明 | 示例 |
|---|---|---|---|---|
| <field> | <string/int/object/array> | 是/否 | 字段语义与约束 | <example> |

## 4. 返回字段
| 字段名 | 类型 | 说明 | 示例 |
|---|---|---|---|
| code | int | 业务状态码 | 0 |
| message | string | 状态说明 | success |
| data | object/array | 业务数据 | {} |

## 5. 错误码
| 错误码 | 含义 | 处理建议 |
|---|---|---|
| <code> | <meaning> | <action> |

## 6. 签名规则
- 是否需要签名：`是/否`
- 签名参与字段：按接口文档列出字段及排序规则。
- 签名算法：如 `MD5` / `HMAC-SHA256`。
- 拼接规则：明确拼接顺序、分隔符、是否包含空值。
- 密钥说明：密钥来源、保管要求、轮换策略。

## 7. 幂等建议
- 明确幂等键（如 `request_id` / `biz_no`）及唯一性范围。
- 约定重复请求的判定窗口与返回策略。
- 对写操作说明“首次成功后重复调用”的一致返回规范。

## 8. 重试建议
- 仅对可重试错误执行重试（网络超时、限流、临时性 5xx）。
- 建议使用指数退避与抖动。
- 明确最大重试次数与超时上限，避免无限重试。
- 对不可重试错误（参数错误、鉴权失败）应立即失败并人工排查。
