# 前端与新手引导模块细分方案（XY-MOD-002）

> 目标验收：**不懂代码用户可独立完成首次部署与首单流程**。
> 范围：安装向导、首次引导、任务式引导、风险操作拦截、帮助中心、诊断包导出。

---

## 2. 六大模块状态机（前端）

> 说明：状态机用于前端控制 UI 流与按钮可用性，不依赖假数据。

### 2.1 安装向导状态机

```text
idle
 → checking_env
   → env_failed (可重试/查看修复)
   → env_ok
     → connecting_account
       → account_failed (重试/更换登录方式)
       → account_ok
         → validating_config
           → config_failed (高亮错误项)
           → config_ok
             → starting_service
               → start_failed (重试/导出诊断)
               → done
```

**关键规则**
- 任何失败态必须给出「下一步可执行动作」。
- “下一步”按钮只在当前步骤 `ok` 时可用。
- 支持断点恢复（刷新后回到最近完成步骤）。

### 2.2 首次引导状态机

```text
pending_consent
 → consented
   → setting_defaults
     → defaults_saved
       → first_syncing
         → sync_partial (允许先进入主控台)
         → sync_done
           → finished
```

**关键规则**
- `sync_partial` 不阻塞进入产品，但在 dashboard 顶部持续提醒。

### 2.3 任务式引导状态机

```text
not_started
 → in_progress
   → blocked (缺前置条件)
   → skipped (用户主动跳过)
   → completed
all_tasks_completed → graduation
```

**关键规则**
- 每个 task 有前置条件/完成条件。
- 跳过后可在任务中心恢复，不丢进度。

### 2.4 风险操作拦截状态机

```text
monitoring_action
 → low_risk (直接执行)
 → medium_risk (二次确认)
 → high_risk (强确认 + 冷静期倒计时)
   → confirmed_execute
   → cancelled
   → denied_by_policy
```

**关键规则**
- 高风险操作需输入确认短语（如“我已知晓风险”）。
- 命中策略拒绝时，提供替代方案入口。

### 2.5 帮助中心状态机

```text
idle
 → searching
   → search_empty (推荐热门问题)
   → search_hit (展示答案卡片)
     → open_article
       → solved
       → unsolved (引导反馈/诊断导出)
```

### 2.6 诊断包导出状态机

```text
idle
 → selecting_scope
   → collecting_logs
     → sanitizing_sensitive_data
       → packaging
         → export_failed (提示空间/权限问题)
         → export_ready (下载/复制路径)
```

**关键规则**
- 默认脱敏：token/cookie/手机号掩码。
- 导出前展示数据范围与隐私说明。

## 3. 关键交互文案（可直接给 UI）

## 3.1 安装向导
- 标题：**“3分钟完成安装，我们会一步步带你走。”**
- 环境检查中：**“正在检查运行环境，请稍候…”**
- 环境失败：**“检测到环境未就绪：{问题项}。按下方指引修复后可一键重试。”**
- 启动成功：**“服务已启动 ✅ 现在开始首次引导。”**

## 3.2 首次引导
- 隐私说明：**“仅采集运行所需信息，你可随时在设置中关闭非必要项。”**
- 首次同步：**“正在准备你的工作台（预计 10~30 秒）”**
- 部分完成：**“核心功能可用，其余数据将在后台继续同步。”**

## 3.3 任务式引导
- 任务卡主标题：**“跟着做 5 个小任务，完成你的首单流程”**
- 阻塞态：**“此任务依赖「{前置任务}」，先完成它再回来。”**
- 完成态：**“太好了！你已完成「{任务名}」。”**

## 3.4 风险操作拦截
- 高风险标题：**“该操作可能导致账号受限，请确认后继续”**
- 冷静期：**“请阅读风险提示（{N} 秒后可继续）”**
- 强确认输入提示：**“请输入：我已知晓风险”**
- 拒绝执行：**“已为你拦截高风险操作。建议改用「安全模式」执行。”**

## 3.5 帮助中心
- 搜索占位：**“输入问题关键词，如：登录失败、首单、环境检查”**
- 未命中：**“暂未找到直接答案，试试这些热门问题”**
- 无法解决：**“仍未解决？一键导出诊断包并提交，我们会更快定位问题。”**

## 3.6 诊断包导出
- 范围说明：**“将包含运行日志、错误栈、版本信息，不包含明文敏感凭据。”**
- 导出中：**“正在整理诊断信息，请勿关闭页面…”**
- 成功：**“诊断包已生成：{fileName}。你可以下载或复制路径给支持人员。”**

## 4. 验收用例（面向小白用户）

## A. 安装向导
1. **A1 正常安装链路**
   - 前置：全新用户，未配置过。
   - 操作：按步骤完成环境检查→账号连接→配置校验→启动。
   - 预期：5步全部通过，显示“安装完成”，可进入首次引导。

2. **A2 环境缺失修复**
   - 前置：模拟某依赖缺失。
   - 操作：进入安装向导。
   - 预期：明确指出缺失项+修复指引；修复后点击“重试”通过。

## B. 首次引导
3. **B1 首次同步成功**
   - 操作：同意隐私→保存默认设置→执行首次同步。
   - 预期：引导完成，进入 dashboard，显示“开始任务式引导”。

4. **B2 同步部分失败不阻塞**
   - 操作：模拟部分数据源失败。
   - 预期：可进入 dashboard，顶部出现“后台继续同步”提醒。

## C. 任务式引导（首单流程核心）
5. **C1 完整首单闭环**
   - 操作：顺序完成 5 个任务（账号→抓取→规则→演练→守护）。
   - 预期：任务中心显示 100%，出现“你已完成首单流程”提示。

6. **C2 跳过与恢复**
   - 操作：在 task-3 点击跳过，后续回到任务中心恢复。
   - 预期：进度正确保留，恢复后可继续完成并达成毕业状态。

## D. 风险操作拦截
7. **D1 高风险拦截有效**
   - 操作：触发高风险动作。
   - 预期：展示风险说明+倒计时+确认短语；未输入短语不可执行。

8. **D2 策略拒绝与替代路径**
   - 操作：触发策略明确禁止动作。
   - 预期：执行被拒绝，提供“安全模式执行”入口。

## E. 帮助中心
9. **E1 搜索直达解决**
   - 操作：搜索“登录失败”。
   - 预期：返回答案卡片，用户标记“已解决”后记录完成。

10. **E2 未解决转诊断**
    - 操作：搜索无结果并点击“仍未解决”。
    - 预期：可一键跳转诊断包导出页面。

## F. 诊断包导出
11. **F1 导出成功**
    - 操作：选择默认范围并导出。
    - 预期：生成文件可下载，显示文件名和生成时间。

12. **F2 脱敏校验**
    - 操作：检查导出包内容。
    - 预期：token/cookie/手机号均被掩码，无明文敏感信息。

---

## 5. 里程碑建议（前端实现顺序）

1. M1：安装向导 + 首次引导骨架（保证可首启）
2. M2：任务式引导 + dashboard任务中心（保证可完成首单）
3. M3：风险拦截 + 帮助中心搜索与文章页
4. M4：诊断包导出 + 全链路验收用例自动化

---

## 6. 完成判定（Definition of Done）

- 新用户从 `/welcome` 开始，无需读代码可走通安装与首单流程。
- 任一步骤失败均有可执行修复动作，不出现“卡死无下一步”。
- 风险操作被有效分级并拦截，高风险无法误触执行。
- 帮助中心与诊断导出形成闭环（自助→求助）。
- 验收用例 A1~F2 全通过。
