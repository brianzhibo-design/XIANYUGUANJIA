## 2) 对比 G3niusYukki 当前实现：领先点与可迁移点

> 对标依据来自 `docs/SUCCESS_BENCHMARK_AND_SURPASS_PLAN.md` 的代码级审查结论，并结合本仓现状。

### 2.1 我方领先点（建议继续强化）

1. **导入与数据治理能力更强**
   - 我方已覆盖 zip 解包、乱码修复、多格式导入与扩展链路；对标侧以“可用”为主，导入维度相对收敛。
2. **首响与标准询价链路更清晰**
   - 我方将“非结构化寒暄”导向结构化询价模板，降低报价前置损耗。
3. **风控/鉴权可观测性更强**
   - 我方将 token、cookie、risk 统一为可读状态与恢复建议；对标侧更偏“运行逻辑可恢复”，但可视化分层较弱。
4. **人工接管机制更工程化**
   - 我方具备会话级持久化开关和统计；对标侧更多依赖控制词与流程约定。
5. **SLA指标化更完整**
   - 我方已沉淀“首响P95/报价成功率/回退率/告警恢复”的运营指标框架。

### 2.2 可迁移点（应吸收）

1. **低门槛运行体验**（对标强项）
   - 一键启动/状态脚本、零心智负担的启动路径。
2. **连接层韧性实现细节**
   - WS 主循环 + 心跳 + 重连闭环在对标项目中实践充分，可持续借鉴其“简单但稳定”的执行模型。
3. **模板化输出的一致性策略**
   - 对标在报价回复格式稳定性上有长期运行经验，可继续吸收其模板约束方法。

## 3) 一页执行清单：核心壁垒 vs 模块化复用

## A. 保留为核心壁垒（不外放/不弱化）

- [ ] **风控状态机与恢复编排**：`token_error + cookie_update_required + risk_control + recovery_advice`
- [ ] **会话级人工接管持久化与跳过策略**：避免人机冲突、保障服务可信度
- [ ] **SLA 观测与告警口径**：首响P95、报价成功率、回退率、恢复耗时统一口径
- [ ] **售前链路的降级策略**：报价失败→人工兜底模板，确保“不失联”

> 原则：这些能力直接决定“可持续成交率”和“运维成本”，属于差异化护城河。

## B. 可模块化复用（可产品化/可插件化）

- [ ] **消息理解插件层**：关键词规则、意图路由、标准询价模板可配置化
- [ ] **报价适配器层**：规则表、远程API、混合模式 provider 接口标准化
- [ ] **导入工具链**：zip/Excel/CSV/OCR 作为独立导入模块对外复用
- [ ] **运行入口与诊断工具**：`start/status/doctor` 可沉淀为通用脚手架
- [ ] **看板组件**：风险/连接/SLA 卡片作为独立 dashboard 组件复用

> 原则：凡是“可配置、低业务耦合、可跨场景复用”的能力，优先模块化。

- **W1（稳态）**：统一口径（SLA、risk、manual）+ 最小E2E证据补齐（真实售前闭环）
