## 1) 本轮执行门禁清单

### A. 覆盖率口径（统一统计规则）
1. **统计范围**：仅统计 `src/` 业务代码。
2. **排除项**：`tests/*`、`test_*.py`、`__pycache__`、`site-packages`（与 `pyproject.toml` 一致）。
3. **统计工具**：`pytest + pytest-cov + coverage`。
4. **全量覆盖率门槛**：`TOTAL >= 35%`（与现有 `pytest.ini` 的 `--cov-fail-under=35` 对齐）。
5. **增量覆盖率口径**：仅统计本轮变更行（相对 `origin/main...HEAD` 的 diff 行）。

### B. pytest 门禁
- 测试必须全部通过，且全量覆盖率不低于 35%。
- 推荐统一命令（可复现）：

```bash
cd '/Users/brianzhibo/Documents/New project/xianyu-openclaw'
python3 -m pip install -r requirements.txt pytest pytest-cov pytest-asyncio coverage
python3 -m pytest tests/ -v --tb=short --cov=src --cov-report=term-missing --cov-report=xml --cov-fail-under=35
```

### C. ruff 门禁
- Lint 必须 0 错误（按仓库规则执行）。
- Format check 必须通过（无待格式化文件）。
- 可复现命令：

```bash
cd '/Users/brianzhibo/Documents/New project/xianyu-openclaw'
python3 -m pip install ruff
python3 -m ruff check src/
python3 -m ruff format --check src/
```

### D. 增量覆盖目标（本轮新增要求）
- **目标阈值**：`diff-cover >= 80%`（本轮修改行至少 80% 被测试覆盖）。
- 计算基线：`origin/main...HEAD`。
- 可复现命令：

```bash
cd '/Users/brianzhibo/Documents/New project/xianyu-openclaw'
python3 -m pip install diff-cover
python3 -m pytest tests/ --cov=src --cov-report=xml --cov-fail-under=35
python3 -m diff_cover coverage.xml --compare-branch=origin/main --fail-under=80
```

> 说明：若本地无 `origin/main`，先执行 `git fetch origin main`。
